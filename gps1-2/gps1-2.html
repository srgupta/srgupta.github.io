Pair Programming Session

What is Pair Programming?

Pair Programming is a technique in software development where two people code on the same project at once. Each person takes a distinct role, either driver or navigator.

Why is pair programming important to DBC?

Pair programming is important to DBC because it has actually become a part of the industry. No one does programming by themselves anymore. It saves time, money and energy to work with someone else as opposed to working by yourself. Errors can be caught much more quickly having another pair of eyes on block of code.
